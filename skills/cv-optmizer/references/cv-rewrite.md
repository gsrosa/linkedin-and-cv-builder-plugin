# Rewrite the full CV

Use when the user wants a **whole-document** pass (summary, experience, skills, ordering) — not only individual bullets. For bullet-level mechanics, use [bullet-analyze.md](bullet-analyze.md) through [bullet-rewrite.md](bullet-rewrite.md).

## When to use

- “Rewrite my CV”, “full resume pass”, “tailor everything for X role”
- After [cv-analyze.md](cv-analyze.md) and [cv-review-strengths-weaknesses.md](cv-review-strengths-weaknesses.md) when a structured rewrite is agreed

## Order of work

1. **Title / headline line** — specific seniority + specialty + optional remote/location signal
2. **Professional summary** — 4–6 lines: who you are, core specialty, proof, what problems you solve (not “seeking opportunities” as the hook)
3. **Experience** — reverse chronological; for each role, tighten bullets using the bullet workflow
4. **Skills** — categorized; aligned with headline and roles; remove keyword stuffing
5. **Education / languages** — short for senior profiles unless academic roles are the goal

## Section rules (summary)

| Section | Focus |
|---------|--------|
| Summary | Identity + specialty + proof; no banned fluff |
| Experience | Action + what + result; metrics where honest |
| Skills | Categories; match job description when tailoring |

Detailed formulas and banned phrases: parent `SKILL.md` (cv-optimizer skill).

## Per-bullet workflow

For each bullet: [bullet-analyze.md](bullet-analyze.md) → [bullet-patterns.md](bullet-patterns.md) → [bullet-rewrite.md](bullet-rewrite.md). Examples: [bullet-rewrites.md](bullet-rewrites.md).

## Output format (full CV rewrite)

Deliver clearly separated sections, e.g.:

1. **Summary** — revised text
2. **Experience** — by company, revised bullets
3. **Skills** — revised groupings
4. **Notes** — global changes (tone, ordering, what was cut)

Keep **unchanged** facts (dates, company names, titles) unless the user corrects them.

## After rewriting

- Offer to save to Notion per parent `SKILL.md` (CV Versions / Current CV)
- Point user to [cv-index.md](cv-index.md) if they want only bullet-level edits next time
