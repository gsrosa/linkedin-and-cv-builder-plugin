# Analyze a single CV bullet

Use for **per-bullet** diagnosis before pattern classification and rewrite. See [cv-index.md](cv-index.md) for the full workflow.

## Objective

Identify weaknesses, missing elements, and improvement opportunities for **one bullet at a time**.

## Evaluate across

### 1. Clarity

- Is the action clearly described?
- Is it specific or vague?

### 2. Ownership

- Does it show ownership or just participation?
- Weak signals: “helped”, “worked on”, “assisted”

### 3. Impact

- Is there a measurable or observable outcome?
- Missing metrics?

### 4. Technical depth

- Does it show real technical decisions?
- Or only tools and buzzwords?

### 5. Context

- Is the problem being solved clear?
- Or only the implementation?

### 6. Seniority signal

- Does it reflect senior-level thinking?
- (trade-offs, architecture, scale, constraints)

## Output (analysis phase)

- Issues found
- Missing elements
- Suggested improvement direction
